# Responsibility

The utils directory serves as the functional core for conflict resolution and state consistency. Its primary responsibilities include:

- Path Validation: Determining if a local or remote path is eligible for synchronization based on its state and metadata.
- Conflict Prevention: Filtering out "unmergeable" paths to prevent data corruption during race conditions.
- Atomic Operations: Consolidating multiple file system tasks (like directory creation or remote deletions) into single, merged operations to minimize I/O and server overhead.
- State Tracking: Providing atomic updates to local records after successful sync cycles.

---

# Design Patterns

The module employs several recurring design patterns to maintain a clean separation of concerns:

- Predicate Pattern: Functions like isMergeablePath and hasIgnoredInFolder act as boolean guards, encapsulating complex logic behind simple interfaces.
- Task Aggregator (Strategy Pattern): The merge-\*-tasks.ts files use an aggregation strategy. Instead of executing many small tasks, they group similar operations (like mkdir or remove-remote) into a single batch, reducing the complexity of the calling function.
- Functional Immutability: The utilities generally treat input data as immutable, returning new task lists or state updates rather than modifying existing objects.

---

# Data & Control Flow

The flow of data through these utilities follows a standard Input → Filter/Merge → Output pipeline:

1.  Input: A list of raw synchronization tasks or file paths.
2.  Validation Loop: \* is-mergeable-path.ts checks if the path is currently in a state that allows merging.
    - has-ignored-in-folder.ts evaluates the path against exclusion rules.
3.  Consolidation: \* merge-mkdir-tasks.ts scans the task list to remove redundant directory creation steps (e.g., creating a parent and its child separately).
    - merge-remove-remote-tasks.ts optimizes the deletion process.
4.  ization: update-records.ts is invoked at the end of a sync cycle to commit the new state to the local database or cache.

---

# Integration Points

These utilities act as the "connective tissue" between the high-level sync engine and the lower-level file system/API adapters:

- Sync Engine (Parent): Calls these utilities to sanitize its task list before execution.
- Local File System: Interacted with via the paths validated by has-ignored-in-folder.
- Remote Vault API: Receives the optimized task lists generated by the merge-\*-tasks utilities.
- Local Storage/Database: Updated via update-records.ts to reflect the result of the sync operation.
